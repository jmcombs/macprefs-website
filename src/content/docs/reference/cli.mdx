## Commands

### `preflight`

Checks write access to user Preferences, runs a safe `defaults` write/delete smoke test, and warns if MDM profiles are present. No permanent changes are made.

```bash
macprefs preflight [--config <config>]
```

| Flag | Description |
|------|-------------|
| `-c, --config <config>` | Path to config JSON (optional, validates if provided) |
| `--version` | Show the version. |
| `-h, --help` | Show help information. |

**Exit codes:**
- `0` — All checks passed
- `2` — One or more issues detected

### `export`

Exports macOS preferences to a JSON configuration file across all domains by default.

```bash

```

**Arguments:**
- `<output-file>` — Output JSON file path (required)

| Flag | Description |
|------|-------------|
| `--domains <domains>` | Comma-separated domains to include (defaults to all domains) |
| `--baseline` | Compare against baseline (v1.1) |
| `--no-progress` | Suppress progress output (for CI/scripting) |
| `--version` | Show the version. |
| `-h, --help` | Show help information. |

### `filter`

Classifies every key in a config using deterministic guardrails (value-facts, name-pattern overrides, and an optional baseline diff) and writes a keep-only config (immediately apply-safe) plus a separate human-readable report of skip/review keys. No AI.

```bash
macprefs filter <input-file> [--output <output>] [--report <report>] [--baseline <baseline>] [--domains <domains>] [--no-progress]
```

**Arguments:**
- `<input-file>` — Input config JSON (export output or hand-authored) (required)

| Flag | Description |
|------|-------------|
| `-o, --output <output>` | Keep-only output config (default: <input>.filtered.json) |
| `--report <report>` | Report path (default: <input>.report.txt) |
| `--baseline <baseline>` | Optional baseline config for changed-from-default detection |
| `--domains <domains>` | Comma-separated domains to include (defaults to all in the input) |
| `--no-progress` | Suppress progress output (for CI/scripting) |
| `--version` | Show the version. |
| `-h, --help` | Show help information. |

**Exit codes:**
- `0` — Completed; no keys require human review (config is apply-safe)
- `2` — Completed, but one or more keys need review

<Aside type="note">
Deterministic and batch — the AI-assisted interactive experience is reserved for the paid GUI (Macprefs Studio).
</Aside>

### `plan`

Shows what changes would be made without applying them. Read-only operation.

```bash
macprefs plan [--config <config>] [--format <format>] [--verbose] [--filter <filter>]
```

| Flag | Description |
|------|-------------|
| `-c, --config <config>` | Path to config JSON |
| `--format <format>` | Output format (table|json) |
| `-v, --verbose` | Show all changes in detail |
| `--filter <filter>` | Filter changes: conflicts, additions, all |
| `--version` | Show the version. |
| `-h, --help` | Show help information. |

**Exit codes:**
- `0` — No changes needed (in sync)
- `2` — Changes detected (drift found)

<Aside type="note">
`--filter` and `--verbose` are mutually exclusive.
</Aside>

### `apply`

Applies config changes. Creates snapshot before any changes.

```bash
macprefs apply [--config <config>] [--yes] [--rollback-on-fail] [--no-rollback-on-fail] [--verbose]
```

| Flag | Description |
|------|-------------|
| `-c, --config <config>` | Path to config JSON |
| `--yes` | Auto-confirm apply (for CI/CD) |
| `--rollback-on-fail` | Rollback on failure (default: enabled) |
| `--no-rollback-on-fail` | Disable rollback on failure |
| `--verbose` | Verbose output |
| `--version` | Show the version. |
| `-h, --help` | Show help information. |

**Exit codes:**
- `0` — Changes applied successfully (or no changes needed)
- `1` — Apply failed

<Aside type="caution">
Without `--yes`, apply prompts for confirmation before applying changes.
</Aside>
